# Pipeline SPEC

Each stage takes an input set, produces an output set, and classifies each
net as PASS (writes a useful file) or FAIL (writes a marker with the same
payload format, so the broken case is inspectable).

Conventions:
- `V` is the vertex count; net identity is its CLERS string `NAME`.
- `data/<stage>/<v>/` is the per-v output directory for each stage.
- Stages are idempotent per net: rerunning with an already-present `.obj`
  output is a skip, not a rewrite. To force a rerun on a net, delete its
  output file(s) first.

## 1. primes

Enumerate all canonical prime 6-net CLERS up to a given `V`.

| in  | out                    |
|-----|------------------------|
|  —  | `data/prime/<v>.txt`   |

Each file has one CLERS per line, canonical form, sorted.

## 2. horodump

Hyperbolic UHS homotopy: solve the finite-ρ system from ρ=4 down to
ρ=2⁻¹⁴, dumping all intermediate frames in binary.

| in                    | out (PASS)           | out (FAIL)              |
|-----------------------|----------------------|-------------------------|
| `data/prime/<v>.txt`  | `data/uhs/<v>/NAME.uhs` | `data/uhs/<v>/NAME.failed` |

`.uhs` binary format:
```
int32 NV
int32 nframes
double frame[0]:   rho=DBL_MAX, then 3*(NV-3) doubles (ideal {x,y,u²})
double frame[k]:   rho,         then 3*(NV-3) doubles ({x,y,t} at ρ=rho)
```
`.failed` is an empty marker file (horodump failure = homotopy didn't
converge via adaptive restart).

## 3. pusheuclid  (push)

Take the last UHS frame, map to Klein, scale by 1/(2ρ) to get Euclidean
coords. Test embedded + undented.

| in                      | out (PASS)                | out (FAIL)                |
|-------------------------|---------------------------|---------------------------|
| `data/uhs/<v>/*.uhs`    | `data/pusheuclid/<v>/NAME.obj` | `data/pusheuclid/<v>/NAME.fail` |

PASS criteria: `check_embedded == 1` AND `min_turn > 0`.

Both PASS and FAIL files contain the full OBJ (v-lines and f-lines). FAIL
files optionally have a `# reason` marker on line 1. A FAIL here means
horodump's last frame didn't land on a valid Euclidean embedding — it's
effectively an undetected horodump failure.

Implementation: `bin/euclidsolve indir outdir < names.txt` (no `-polish`).

## 4. polish

Take the pushed `.obj` and drive all edge lengths to 1.

| in                         | out (PASS)             | out (FAIL)             |
|----------------------------|------------------------|------------------------|
| `data/pusheuclid/<v>/*.obj`| `data/polish/<v>/NAME.obj` | `data/polish/<v>/NAME.fail` |

Internally two solvers, raced by fallback:

- **polishA** — Newton on the edge-length system with residual
  backtracking, gauge pin v1=(0,0,0), v2=(1,0,0), v3=(√3/2, ½, 0). Target
  max|edge_len²−1| < 10⁻¹⁰ in ≤ 50 iters.
- **polishB** — augmented Newton. Triggered when polishA stalls (final
  max|r| ≥ 10⁻¹⁰). Identify flat deg-6 vertices (|turn| < 10⁻⁶). For each
  flat v, pick 2 link neighbors `keep1, keep2` (flat neighbors preferred).
  Replace each length equation (v, o) where o ∈ link(v) \ {keep1, keep2}
  with a coplanarity equation `det(o−v, keep1−v, keep2−v) = 0`. Per-edge
  dedup when adjacent flats share a drop. Run Newton on the augmented
  square system.

PASS criteria: (polishA converged OR polishB converged) AND `check_embedded`.

Both PASS and FAIL files contain the full OBJ. FAIL files get a
`# reason rA=.. rB=..` marker.

Incremental: if `data/polish/<v>/NAME.obj` already exists, the net is
skipped entirely (no-op, not counted in fail tally).

Implementation: `bin/polish indir outdir < names.txt`.

## 5. prove (dispatch)

Classify the polished net by `min_turn`, then run the right prover.

| in                       | route                                  |
|--------------------------|----------------------------------------|
| `data/polish/<v>/*.obj` with `min_turn` > 10⁻⁶  | Plan A (`bin/prove`) |
| `data/polish/<v>/*.obj` with `min_turn` ≤ 10⁻⁶  | Plan B (C port; TODO) |

Outputs to `data/proofs/<v>_float.txt` (and `failures.txt` summary) as
the existing prover does. The `min_turn` threshold separates rigid /
near-flopper from true flopper.

### Preapproved shortcut

If `NAME` appears in `floppy/data/preapproved/<v>.txt` (generated by
`floppy/src/gen_preapproved.py`), the net is a winner-by-construction —
no prover needed. The preapproved list covers pancakes, octahedra
(no-deg-3), thick pancakes, the hex antiprism, and compounds built from
these as donors.

## Summary of invariants

- `data/prime/<v>.txt` is the source of truth for which nets exist.
- Each stage's output `.obj` is **the** polished geometry at that stage.
- Extensions: `.obj` = PASS, `.fail` = FAIL (with OBJ body), `.failed` =
  stage-specific marker (currently only horodump). Mixing is a bug.
- All pass/fail files contain enough data to reproduce downstream work.
- Stages never modify an existing `.obj` output; to redo, delete first.
